# ExpGitFile 用户手册

## 插件简介

ExpGitFile 是一款高效的 Git 文件导出工具，帮助开发者快速提取代码变更，简化部署流程。支持导出已提交和未提交的文件变更，并提供智能过滤、文件预览、配置管理等功能。

## 安装

1. 打开 IDEA → Settings → Plugins
2. 搜索 "ExpGitFile" 或手动安装插件包
3. 重启 IDEA
4. 入口：Tools → ExpGitSubmitFile

## 功能特性

### 1. 已提交文件导出

选择 Git 提交记录，导出该提交中变更的文件。

- **快速选择**：支持最近N个提交、今日提交、本周提交
- **自定义选择**：通过提交选择对话框精确选择
- **批量导出**：支持同时导出多个提交的文件

### 2. 未提交文件导出

导出当前工作区中未提交的变更文件。

### 3. 智能过滤规则

支持预设规则和自定义规则：

| 预设规则 | 说明 |
|---------|------|
| 仅Java源码 | 只导出 .java 文件 |
| 排除前端资源 | 排除 .js/.css/.html 等 |
| 后端API文件 | 导出 Controller/Service 等后端文件 |
| 配置文件 | 导出 .xml/.properties/.yml 等 |
| 排除测试文件 | 排除 test 目录下的文件 |
| Web资源 | 导出静态资源文件 |
| SQL文件 | 导出 .sql 文件 |

### 4. 文件预览

导出前可预览文件列表，支持：
- 查看文件差异对比
- 选择部分文件导出
- 在编辑器中打开文件

### 5. 配置管理

- **保存配置**：将当前设置保存为模板
- **加载配置**：一键加载已保存的配置
- **导出历史**：查看历史导出记录，支持重新执行

### 6. 高级选项

| 选项 | 说明 |
|-----|------|
| 显示文件预览 | 导出完成后自动刷新文件预览面板 |
| 过滤测试文件 | 自动排除测试相关文件 |
| 包含空提交 | 导出时包含没有文件变更的提交 |
| 保留文件结构 | 保持原有的目录结构 |
| 创建导出日志 | 生成导出操作的日志文件 |

## 使用流程

### 导出已提交文件

1. 设置 Git 仓库路径
2. 设置目标导出路径
3. 选择"已提交"类型
4. 选择提交范围或自定义选择提交记录
5. （可选）设置过滤规则
6. 点击"开始导出"

### 导出未提交文件

1. 设置 Git 仓库路径
2. 设置目标导出路径
3. 选择"未提交"类型
4. （可选）设置过滤规则
5. 点击"开始导出"

## 常见问题

### Q: 导出时提示"提交的SHA-1哈希值不能为空"？

A: 已提交模式下必须选择提交记录，请点击"选择提交..."按钮选择要导出的提交。

### Q: 导出的文件路径不正确？

A: 请确保 Git 仓库路径设置正确，插件会自动识别项目中的 Git 仓库。

### Q: 如何只导出特定类型的文件？

A: 使用"设置过滤规则"功能，选择预设规则或创建自定义规则。

### Q: 导出历史在哪里查看？

A: 在插件窗口中切换到"导出历史"标签页查看。
